# time-overlay

`time-overlay` is a Next.js app bootstrapped with `pnpm`, `TypeScript`, `ESLint`, App Router, Tailwind CSS v4, `src/` layout, and `shadcn/ui`.

## Stack

- Next.js 16 App Router
- React 19
- TypeScript 5
- Tailwind CSS 4
- `shadcn/ui` with `base-nova` style and `@base-ui/react`
- `lucide-react` for icons

## Commands

```bash
pnpm dev
pnpm build
pnpm start
pnpm lint
pnpm test -- --run
```

## Project Structure

```text
src/
  app/              App Router entry, layouts, pages, and global styles
  components/
    ui/             shadcn/ui primitives added through the shadcn CLI
  lib/              shared utilities such as cn()
```

## Development Conventions

### General

- Use `pnpm` for all package and script operations.
- Keep route files under `src/app`.
- Use the `@/*` import alias instead of long relative paths.
- Prefer React Server Components by default. Add `"use client"` only when a file needs client-only hooks, browser APIs, or event handlers.

### UI and Styling

- Prefer `shadcn/ui` primitives before writing custom UI wrappers.
- Add new primitives with `pnpm dlx shadcn@latest add <component>`.
- This project uses the `shadcn/ui` `base` foundation, not `radix`. Where composition APIs differ, prefer `render` over `asChild`.
- Use the existing aliases from `components.json`: `@/components`, `@/components/ui`, `@/lib`, `@/hooks`.
- Keep styling in Tailwind utility classes and semantic tokens from `src/app/globals.css`.
- Reuse the `cyber-*` utilities from `src/app/globals.css` for the public shell and homepage surfaces before introducing new long-form effect class stacks. The shared primitives are `cyber-panel`, `cyber-panel-muted`, `cyber-chamfer`, `cyber-grid`, `cyber-scanlines`, and `cyber-glitch`.
- Avoid raw one-off color utilities when an existing semantic token or component variant already fits.
- Use `gap-*` for spacing instead of `space-x-*` or `space-y-*`.

### Components

- Put reusable app-specific components in `src/components`.
- Keep `src/components/ui` for code generated by `shadcn/ui` or small local adjustments to generated primitives.
- Prefer composing existing primitives instead of creating large custom abstractions early.
- If width and height are equal, prefer Tailwind `size-*`.

### Workflow

- When adding UI, check whether the component already exists in `src/components/ui` before creating new markup.
- After adding a new `shadcn/ui` component, verify imports and keep the project on the installed `base-nova` style.
- Run `pnpm lint` before considering a change complete.

### SEO and Public Pages

- Public routes should live in `src/app/<segment>/page.tsx`.
- Export route metadata with `createPageMetadata(...)` from `src/lib/site.ts` so title, description, canonical, Open Graph, and Twitter fields stay aligned.
- Keep one visible H1 per public page and prefer plain `Link`-based internal navigation with descriptive anchor text.
- When adding a new public page, decide whether it belongs in `siteConfig.navItems` and add any homepage cross-links in the relevant `src/components/site/*` modules.

## Notes

- `components.json` is the source of truth for `shadcn/ui` configuration.
- `src/app/globals.css` is the shared place for global Tailwind and theme tokens.
- `/` is the tool-first homepage and the primary SEO surface.
- `/` is the only public route surface. Generator UX and SEO context both live on the root page.
- The MVP export strategy is client-side and local-first: prioritize `PNG sequence` and `WebM`, and avoid paid server rendering on Vercel.
